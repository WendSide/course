Актуальность темы

В современном мире информационных технологий рынок купли-продажи автомобилей активно перемещается в онлайн-пространство. По данным аналитических агентств, более 70% потенциальных покупателей автомобилей начинают поиск с интернет-площадок, а не с посещения автосалонов. Ежегодно миллионы людей по всему миру используют электронные доски объявлений для поиска и продажи транспортных средств.

Традиционные методы покупки автомобиля через газеты, физические доски объявлений или салоны постепенно уходят в прошлое, уступая место цифровым платформам. Онлайн-площадки предлагают более широкий выбор, удобный поиск с фильтрацией по множеству параметров и возможность сравнения десятков предложений за считанные минуты. Кроме того, цифровые платформы значительно снижают транзакционные издержки: продавец тратит меньше времени и денег на рекламу, а покупатель экономит время на поездках по авторынкам.

Электронные площадки для размещения объявлений о продаже транспортных средств становятся неотъемлемой частью автомобильной инфраструктуры. Такие сервисы позволяют продавцам быстро находить покупателей, минимизируя временные и финансовые затраты на рекламу, а покупателям — удобно просматривать актуальные предложения, фильтровать их по различным критериям и связываться с продавцами напрямую. Дополнительным преимуществом онлайн-площадок является возможность размещения фотографий, что позволяет потенциальному покупателю оценить внешнее состояние автомобиля еще до личного осмотра.

Особую значимость приобретают специализированные автомобильные каталоги, которые предоставляют не просто список объявлений, а структурированную информацию с техническими характеристиками, историей обслуживания и возможностью сравнения моделей. Такие сервисы становятся незаменимыми помощниками как для частных лиц, так и для профессиональных перекупщиков и автосалонов.

Глава 1. Использование технологий

1.1. Обзор технологического стека

При разработке веб-приложения «Sophisticated Motors» был выбран классический стек технологий LAMP (Linux, Apache, MySQL, PHP) с клиентской частью на нативном JavaScript. Данный выбор обусловлен несколькими факторами.

Во-первых, стек LAMP является одним из самых распространенных и хорошо документированных решений в веб-разработке. Согласно статистике W3Techs, более 60% всех веб-сайтов используют PHP, а MySQL занимает лидирующие позиции среди СУБД для веб-приложений. Это обеспечивает доступность информации, обширное сообщество разработчиков и множество готовых решений для типовых задач.

Во-вторых, все компоненты стека являются бесплатными и с открытым исходным кодом, что особенно важно при разработке учебных и некоммерческих проектов. Отсутствие лицензионных отчислений позволяет свободно распространять и модифицировать приложение.

В-третьих, сочетание PHP и MySQL обеспечивает высокую производительность при работе с базами данных и простоту реализации серверной логики. PHP имеет встроенную поддержку работы с MySQL через расширения MySQLi и PDO, что упрощает написание безопасного и эффективного кода.

В-четвертых, нативный JavaScript (без фреймворков) был выбран для:

· минимизации объема кода и избежания избыточных зависимостей (общий объем JavaScript-кода составляет менее 500 строк);
· упрощения отладки и понимания кода для других разработчиков;
· обеспечения более высокой производительности за счет отсутствия дополнительных слоев абстракции (React или Vue добавляют 30-100 КБ к размеру страницы);
· сохранения полного контроля над всеми аспектами работы приложения.

В качестве серверного программного обеспечения на этапе разработки использовался встроенный веб-сервер PHP (php -S localhost:8000), однако для промышленной эксплуатации рекомендуется использование Apache или Nginx с модулем PHP-FPM. Такой подход обеспечивает высокую масштабируемость и возможность обработки большого количества параллельных запросов (до 1000+ запросов в секунду при правильной настройке).

1.2. Frontend технологии

1.2.1. HTML5

HTML5 (HyperText Markup Language version 5) — язык гипертекстовой разметки пятого поколения, который используется для создания структуры веб-страниц. В рамках данного проекта HTML5 применялся для следующих целей:

Создание семантической структуры. Использованы современные семантические элементы:

· <header> — шапка сайта с логотипом и навигационным меню;
· <nav> — блок навигационных ссылок;
· <main> — основное содержимое страницы;
· <section> — группировка тематического контента (блок популярных марок, слайдер);
· <footer> — подвал с информацией об авторских правах;
· <form> — формы регистрации и размещения объявлений.

Применение семантической верстки улучшает:

· доступность — скринридеры (программы для незрячих пользователей) лучше интерпретируют структуру страницы;
· SEO (поисковую оптимизацию) — поисковые системы лучше понимают значимость контента;
· поддерживаемость кода — другим разработчикам легче ориентироваться в структуре.

Реализация форм с встроенной валидацией. Формы регистрации и размещения объявлений используют HTML-атрибуты для базовой проверки данных до отправки на сервер:

· required — обязательное поле;
· minlength="12" — минимальная длина текста;
· type="email" — проверка формата email;
· type="number" — ввод только чисел;
· accept="image/jpeg,image/png" — фильтрация типов файлов.

Это снижает нагрузку на сервер (некорректные запросы отсекаются на клиенте) и улучшает пользовательский опыт (мгновенная обратная связь без перезагрузки страницы).

Организация загрузки файлов. Элемент <input type="file"> с атрибутом accept ограничивает выбор пользователя только допустимыми форматами изображений (JPEG, PNG, JPG). На сервере выполняется дополнительная проверка для безопасности.

Адаптивная верстка. Мета-тег <meta name="viewport" content="width=device-width, initial-scale=1.0"> обеспечивает корректное масштабирование страницы на мобильных устройствах. Без этого тега мобильные браузеры отображают страницу как на десктопе, делая текст и кнопки слишком мелкими.

CSS3 (Cascading Style Sheets version 3) — каскадные таблицы стилей третьего поколения, отвечающие за визуальное оформление веб-страниц. Общий объем CSS-кода составляет более 800 строк, включая стили для всех страниц и состояний.

1.2.3. JavaScript (ES6+)

JavaScript — язык программирования, отвечающий за динамическое поведение веб-страниц. В проекте используется современный стандарт ES6+ (ECMAScript 2015 и новее), включающий стрелочные функции, async/await, классы, шаблонные строки и другие возможности.

Асинхронная загрузка данных (Fetch API). Функция loadCars() отправляет GET-запрос к серверному скрипту и обрабатывает ответ:

```javascript
async function loadCars() {
    try {
        const response = await fetch('http://localhost/myserver/carpostv2.php');
        if (!response.ok) throw new Error('Ошибка HTTP: ' + response.status);
        const cars = await response.json();
        renderCars(cars);
    } catch (error) {
        console.error('Ошибка загрузки:', error);
        document.getElementById('carsContainer').innerHTML = '<p>Ошибка загрузки данных</p>';
    }
}
```

Такой подход (AJAX) позволяет обновлять содержимое страницы без её перезагрузки, что улучшает пользовательский опыт и снижает нагрузку на сервер.

Динамическое создание DOM-элементов. JavaScript формирует HTML-код карточек на основе полученных данных и вставляет его в контейнер:

```javascript
function renderCars(cars) {
    const container = document.getElementById('carsContainer');
    container.innerHTML = cars.map(car => `
        <div class="car-item" data-id="${car.id}">
            <div class="car-image">${getPhotoHtml(car.photo)}</div>
            <div class="car-info">
                <h2>${car.mark} ${car.model}</h2>
                <div class="car-price">${Number(car.price).toLocaleString()} ₽</div>
                <!-- остальные поля -->
            </div>
        </div>
    `).join('');
}
```

Обработка отправки форм. При отправке формы размещения объявления событие submit перехватывается, выполняется клиентская валидация, затем формируется объект FormData и отправляется на сервер методом POST.

Модальные окна. При клике на изображение динамически создается элемент с классом modal-overlay, внутри которого размещается увеличенная копия фотографии. Реализованы несколько способов закрытия: клик на фон, кнопка закрытия, клавиша ESC.

Слайдер. Управление переключением слайдов включает:

· автоматическую прокрутку каждые 5 секунд;
· ручное управление кнопками «вперед/назад»;
· индикацию текущего слайда с помощью точек;
· плавный переход благодаря CSS-свойству transition: transform 0.5s ease.

Работа с localStorage. Функционал избранного использует локальное хранилище браузера для сохранения идентификаторов понравившихся объявлений:

```javascript
function addToFavourites(carId) {
    let favourites = JSON.parse(localStorage.getItem('favourites') || '[]');
    if (!favourites.includes(carId)) {
        favourites.push(carId);
        localStorage.setItem('favourites', JSON.stringify(favourites));

```

1.3. Backend технологии

1.3.1. PHP

Серверная часть приложения реализована на языке PHP (Hypertext Preprocessor) версии 7.4 и выше. PHP был выбран по следующим причинам:

· Простота изучения и использования — синтаксис PHP интуитивно понятен разработчикам, знакомым с C-подобными языками (Java, C++, JavaScript).
· Встроенная поддержка работы с формами — PHP автоматически обрабатывает данные POST и GET, а также загруженные файлы через суперглобальные массивы $_POST, $_GET, $_FILES. Это избавляет разработчика от необходимости писать сложные парсеры.
· Отличная интеграция с MySQL — расширения MySQLi и PDO обеспечивают удобное и безопасное взаимодействие с базой данных. PDO поддерживает параметризованные запросы, полностью исключающие SQL-инъекции.
· Кроссплатформенность — PHP-скрипты могут выполняться на различных операционных системах (Windows, Linux, macOS) без изменения кода. Это позволяет разрабатывать приложение на локальной машине с Windows и развертывать на Linux-сервере.
· Огромное сообщество — на любой вопрос по PHP можно найти ответ на Stack Overflow или в официальной документации.

В рамках проекта созданы два основных PHP-скрипта:

1. carpostv2.php — скрипт для получения списка автомобилей (GET-запрос)

Алгоритм работы:

1. Устанавливает CORS-заголовки для разрешения кросс-доменных запросов.
2. Подключается к базе данных через PDO.
3. Выполняет SELECT-запрос ко всем полям таблицы carpost.
4. Преобразует результат в ассоциативный массив.
5. Обрабатывает поле photo (замена NULL на пустую строку).
6. Возвращает JSON-ответ с помощью json_encode().

2. post.php — скрипт для добавления нового объявления (POST-запрос)

Алгоритм работы:

1. Проверяет метод запроса (должен быть POST).
2. Подключается к базе данных.
3. Извлекает данные из $_POST и $_FILES.
4. Обрабатывает загруженный файл:
   · создает директорию uploads/ при необходимости;
   · проверяет расширение файла (jpg, jpeg, png, gif, webp);
   · генерирует уникальное имя (timestamp + случайное число);
   · перемещает файл из временной директории.
5. Выполняет валидацию обязательных полей.
6. Формирует параметризованный SQL-запрос INSERT.
7. Связывает параметры с переменными через bindParam().
8. Выполняет запрос и возвращает JSON-ответ с результатом.

Листинг 1.1 – Пример параметризованного запроса (защита от SQL-инъекций)

```php
// НЕБЕЗОПАСНО (уязвимо для SQL-инъекций):
$sql = "INSERT INTO carpost (mark) VALUES ('" . $_POST['mark'] . "')";

// БЕЗОПАСНО (параметризованный запрос):
$sql = "INSERT INTO carpost (mark) VALUES (:mark)";
$stmt = $pdo->prepare($sql);
$stmt->bindParam(':mark', $_POST['mark']);
$stmt->execute();
```

1.3.2. Обработка ошибок

В PHP-скриптах реализована обработка исключений с использованием конструкции try-catch:

```php
try {
    $pdo = new PDO("mysql:host=$host;dbname=$db;charset=utf8", $user, $pass);
    $pdo->setAttribute(PDO::ATTR_ERRMODE, PDO::ERRMODE_EXCEPTION);
} catch (PDOException $e) {
    echo json_encode(['error' => 'Ошибка БД: ' . $e->getMessage()]);
    exit();
}
```

В случае возникновения ошибки (например, проблемы подключения к БД) скрипт возвращает HTTP-код 500 и сообщение об ошибке в формате JSON. Для production-среды рекомендуется логировать ошибки в файл, а пользователю показывать общее сообщение «Произошла ошибка».

1.4. Система управления базами данных

В качестве системы управления базами данных (СУБД) используется MySQL версии 5.7 и выше. Выбор MySQL обусловлен следующими факторами:

· Высокая производительность — MySQL оптимизирована для работы с большими объемами данных и высокими нагрузками. При правильной настройке индексов запросы выполняются за миллисекунды.
· Надежность и стабильность — MySQL используется в крупнейших интернет-проектах (Facebook, Twitter, YouTube, Wikipedia) и доказала свою способность работать 24/7 без сбоев.
· Поддержка транзакций — с использованием движка InnoDB MySQL поддерживает ACID-совместимые транзакции (атомарность, согласованность, изоляция, долговечность), что важно для целостности данных. Например, при добавлении объявления и фотографии можно использовать транзакцию, чтобы обе операции выполнились успешно или ни одна.
· Простота администрирования — для управления базами данных существует множество инструментов: phpMyAdmin (веб-интерфейс), MySQL Workbench (графическое приложение), командная строка.
· Бесплатность — MySQL Community Edition распространяется под лицензией GPL, что делает её доступной для любых проектов, включая коммерческие.

2.3. Описание серверной части

2.3.1. Скрипт carpostv2.php

Полный листинг с подробными комментариями:

```php
<?php
// Отключаем вывод ошибок для production-среды (на этапе разработки можно включить)
error_reporting(0);
ini_set('display_errors', 0);

// Устанавливаем CORS-заголовки для разрешения кросс-доменных запросов
header('Access-Control-Allow-Origin: *');
header('Access-Control-Allow-Methods: GET, POST, OPTIONS');
header('Access-Control-Allow-Headers: Content-Type');
header('Content-Type: application/json');

// Параметры подключения к базе данных
$host = 'localhost';                 // Хост БД (обычно localhost)
$db   = 'projectbaseautosell';       // Имя базы данных
$user = 'root';                      // Пользователь БД
$pass = '';                          // Пароль (пустой для локальной разработки)

try {
    // Создаем подключение через PDO с кодировкой UTF-8
    $pdo = new PDO("mysql:host=$host;dbname=$db;charset=utf8", $user, $pass);
    // Устанавливаем режим ошибок: исключения
    $pdo->setAttribute(PDO::ATTR_ERRMODE, PDO::ERRMODE_EXCEPTION);
    // Устанавливаем режим выборки: ассоциативный массив
    $pdo->setAttribute(PDO::ATTR_DEFAULT_FETCH_MODE, PDO::FETCH_ASSOC);
} catch (PDOException $e) {
    // В случае ошибки подключения возвращаем JSON с описанием
    echo json_encode(['error' => 'Ошибка БД: ' . $e->getMessage()]);
    exit();
}

// Выполняем SELECT-запрос для получения всех полей из таблицы carpost
$stmt = $pdo->query("SELECT id, mark, model, `release`, price, mileage, ecapacity, 
                     transmission, `condition`, numberowners, description, sity, 
                     numberphone, owner, photo FROM carpost");
$cars = $stmt->fetchAll();

// Обрабатываем поле photo: если значение NULL, заменяем на пустую строку
foreach ($cars as &$car) {
    if (!isset($car['photo']) || $car['photo'] === null) {
        $car['photo'] = '';
    }
}

// Очищаем буфер вывода (на случай, если были неявные пробелы или предупреждения)
ob_clean();
// Отправляем JSON-ответ
echo json_encode($cars);
exit();
?>
```

2.3.2. Скрипт post.php

Ключевые части скрипта (обработка файла и валидация):

```php
<?php
// Включаем вывод ошибок для отладки (на продакшне отключить)
error_reporting(E_ALL);
ini_set('display_errors', 1);

// ... CORS-заголовки (аналогично carpostv2.php) ...

if ($_SERVER['REQUEST_METHOD'] === 'POST') {
    // Подключение к БД (аналогично carpostv2.php)
    
    // Получение данных из POST
    $mark = $_POST['mark'] ?? '';
    $model = $_POST['model'] ?? '';
    // ... остальные поля ...
    
    // Обработка загруженного файла
    $photo_path = null;
    if (isset($_FILES['photo']) && $_FILES['photo']['error'] === UPLOAD_ERR_OK) {
        $upload_dir = 'uploads/';
        if (!file_exists($upload_dir)) {
            mkdir($upload_dir, 0777, true);
        }
        
        $ext = strtolower(pathinfo($_FILES['photo']['name'], PATHINFO_EXTENSION));
        $allowed = ['jpg', 'jpeg', 'png', 'gif', 'webp'];
        
        if (in_array($ext, $allowed)) {
            $filename = time() . '_' . rand(1000, 9999) . '.' . $ext;
            $photo_path = $upload_dir . $filename;
            move_uploaded_file($_FILES['photo']['tmp_name'], $photo_path);
        }
    }
    
    // Валидация обязательных полей
    if (empty($mark) || empty($model) || empty($release) || empty($price) || 
        empty($condition) || empty($sity) || empty($numberphone) || empty($owner)) {
        http_response_code(400);
        echo json_encode(['success' => false, 'message' => 'Заполните все обязательные поля']);
        exit();
    }
    
    // Параметризованный запрос
    $sql = "INSERT INTO carpost (mark, model, `release`, price, mileage, ecapacity, 
            transmission, `condition`, numberowners, description, sity, numberphone, owner, photo) 
            VALUES (:mark, :model, :release, :price, :mileage, :ecapacity, :transmission, 
            :condition, :numberowners, :description, :sity, :numberphone, :owner, :photo)";
    
    $stmt = $pdo->prepare($sql);
    $stmt->bindParam(':mark', $mark);
    $stmt->bindParam(':model', $model);
    // ... привязка остальных параметров ...
    $stmt->bindParam(':photo', $photo_path);
    $stmt->execute();
    
    echo json_encode(['success' => true, 'message' => 'Объявление добавлено']);
}
?>
```

2.4. Взаимодействие с базой данных

Для взаимодействия с MySQL используется расширение PDO (PHP Data Objects), которое предоставляет унифицированный интерфейс для работы с различными базами данных. Преимущества PDO:

1. Безопасность. PDO поддерживает параметризованные запросы, которые автоматически экранируют спецсимволы, полностью исключая возможность SQL-инъекций.
2. Переносимость. PDO имеет драйверы для большинства СУБД (MySQL, PostgreSQL, SQLite, Oracle, MSSQL), что позволяет сменить базу данных с минимальными изменениями кода.
3. Объектно-ориентированный интерфейс. Работа с PDO интуитивно понятна для PHP-разработчиков.

Пример подключения с обработкой ошибок:

```php
try {
    $pdo = new PDO("mysql:host=$host;dbname=$db;charset=utf8", $user, $pass);
    $pdo->setAttribute(PDO::ATTR_ERRMODE, PDO::ERRMODE_EXCEPTION);
    $pdo->setAttribute(PDO::ATTR_DEFAULT_FETCH_MODE, PDO::FETCH_ASSOC);
} catch (PDOException $e) {
    die("Ошибка подключения: " . $e->getMessage());
}
```

Пример безопасного SELECT с параметром:

```php
$sql = "SELECT * FROM carpost WHERE mark = :mark";
$stmt = $pdo->prepare($sql);
$stmt->bindParam(':mark', $mark);
$stmt->execute();
$result = $stmt->fetchAll();
```

2.6. Система безопасности и защита данных

В приложении реализованы следующие меры безопасности:

Защита от SQL-инъекций:

· Использование параметризованных запросов PDO.
· Отсутствие прямой конкатенации пользовательских данных в SQL-запросах.

Защита от XSS (межсайтового скриптинга):

· Экранирование выводимых данных с помощью HTML-спецсимволов (функция htmlspecialchars() на сервере, хотя в текущей версии данные выводятся через innerHTML, что потенциально опасно, рекомендуется использовать textContent или экранирование).

Защита при загрузке файлов:

· Проверка расширения файла (только jpg, jpeg, png, gif, webp).
· Генерация уникального имени файла (timestamp + random), исключающая перезапись существующих файлов и path traversal атаки.
· Проверка кода ошибки загрузки (UPLOAD_ERR_OK).
· Хранение файлов в отдельной директории uploads/, недоступной для прямого выполнения PHP-скриптов.

Валидация входных данных:

· На клиенте: проверка обязательных полей, форматов email, длины пароля.
· На сервере: повторная проверка обязательных полей, приведение типов.

Рекомендации по усилению безопасности (для production-версии):

· Использовать HTTPS для шифрования трафика.
· Хранить пароли в хешированном виде (password_hash(), password_verify()).
· Добавить CSRF-токены для защиты форм от межсайтовых поддельных запросов.
· Ограничить размер загружаемых файлов (через PHP-директиву upload_max_filesize).
· Настроить .htaccess для запрета доступа к директории uploads/ напрямую.
· Использовать подготовленные выражения PDO для всех запросов к БД.

3.4. Избранное и регистрация

3.4.1. Система избранного

Избранное реализовано с использованием localStorage — механизма браузера для хранения небольших объемов данных (до 5-10 МБ). Принцип работы:

1. При клике на кнопку «В избранное» (звездочка) идентификатор автомобиля сохраняется в массив favourites в localStorage.
2. На странице love.html этот массив считывается.
3. Загружаются все объявления с сервера через carpostv2.php.
4. Объявления фильтруются по ID, оставляя только те, что есть в избранном.
5. Отображаются отфильтрованные объявления.
6. Пользователь может удалять объявления из избранного — это также обновляет localStorage и интерфейс.

Пример кода для добавления в избранное:

```javascript
function addToFavourites(carId) {
    let favourites = JSON.parse(localStorage.getItem('favourites') || '[]');
    if (!favourites.includes(carId)) {
        favourites.push(carId);
        localStorage.setItem('favourites', JSON.stringify(favourites));
        alert('Добавлено в избранное');
    } else {
        alert('Уже в избранном');
    }
}
```

3.4.2. Система регистрации

Форма регистрации содержит валидацию на стороне клиента. В текущей версии данные не отправляются на сервер, но подготовлена структура для интеграции с таблицей regbase.

Поля формы регистрации:

· Логин (nickname) — минимум 12 символов, только буквы и цифры.
· Email — проверка формата через регулярное выражение.
· Пароль (pass) — минимум 8 символов.
· Подтверждение пароля — должен совпадать с паролем.
· Чекбокс согласия с пользовательским соглашением.

Регулярное выражение для проверки логина:

```javascript
const nicknameRegex = /^[a-zA-Z0-9]{12,}$/;
```

Это выражение означает: строка от 12 символов, состоящая только из латинских букв (a-z, A-Z) и цифр (0-9).

Рекомендации по доработке системы регистрации:

· Хранить пароли в хешированном виде (password_hash).
· Добавить проверку уникальности логина и email.
· Реализовать сессии или JWT-токены для аутентификации.
· Добавить возможность восстановления пароля через email.

Разработка проекта начиналась с этапа планирования, который включал следующие шаги:

1. Анализ предметной области. Были изучены существующие аналоги:

· Auto.ru — крупнейшая автомобильная площадка в России с расширенными фильтрами, историей цен, отзывами.
· Avito Авто — раздел на популярной доске объявлений с простым интерфейсом.
· Дром — специализированный автомобильный портал с форумом и новостями.
· CarGurus — международная платформа с анализом рыночной цены.

На основе анализа были выделены ключевые функции, необходимые для минимально жизнеспособного продукта (MVP):

· Просмотр списка объявлений.
· Просмотр подробной информации об автомобиле.
· Размещение объявления с фотографией.
· Фильтрация по марке.
· Избранное.

2. Сбор требований. Определены функциональные и нефункциональные требования.

Функциональные требования:

· FR1: Приложение должно отображать список всех автомобилей из базы данных.
· FR2: Каждая карточка автомобиля должна содержать фото, марку, модель, цену и основные характеристики.
· FR3: Пользователь должен иметь возможность добавить новое объявление через форму.
· FR4: Форма добавления должна включать обязательные поля и валидацию.
· FR5: При добавлении объявления должна быть возможность загрузить фотографию.
· FR6: Пользователь должен иметь возможность сохранять объявления в избранное.
· FR7: Избранные объявления должны сохраняться между сессиями.
· FR8: Приложение должно предоставлять форму регистрации с валидацией.
· FR9: Главная страница должна содержать слайдер с изображениями.

Нефункциональные требования:

· NFR1: Время загрузки главной страницы не более 2 секунд.
· NFR2: Приложение должно корректно отображаться на устройствах с шириной экрана от 320px до 1920px.
· NFR3: Приложение должно работать в браузерах последних двух поколений.
· NFR4: Код должен быть защищен от SQL-инъекций.
· NFR5: Интерфейс должен быть интуитивно понятным.

3. Выбор технологий. Обоснован выбор стека LAMP + нативный JavaScript (подробно в Главе 1).

4. Составление плана работ. Разработка разбита на этапы с указанием сроков (подробно в разделе 4.7).

Этап проектирования базы данных включал логическое и физическое проектирование.

Логическое проектирование:

Определены следующие сущности и их атрибуты:

Сущность Автомобиль (Объявление):

· Марка (строка, обязательное)
· Модель (строка, обязательное)
· Год выпуска (целое число, обязательное)
· Цена (десятичное число, обязательное)
· Пробег (целое число, опционально)
· Объем двигателя (строка, опционально)
· Коробка передач (строка, опционально)
· Состояние (строка, обязательное)
· Количество владельцев (строка, опционально)

· Имя продавца (строка, обязательное)
· Фотография (текст, опционально)
· Дата создания (дата и время, автоматически)

Сущность Пользователь:

· Логин (строка, обязательное, уникальное)

Связи: Один пользователь может иметь много объявлений (связь один-ко-многим).

Физическое проектирование (SQL-скрипт создания таблиц):

```sql
-- Создание базы данных
CREATE DATABASE IF NOT EXISTS `projectbaseautosell` 
CHARACTER SET utf8mb4 COLLATE utf8mb4_general_ci;

USE `projectbaseautosell`;

-- Таблица объявлений
CREATE TABLE `carpost` (
    `id` INT(11) NOT NULL AUTO_INCREMENT,
    `mark` VARCHAR(100) NOT NULL,
    `model` VARCHAR(100) NOT NULL,
    `release` INT(11) NOT NULL,
    `price` DECIMAL(15,2) NOT NULL,
    `mileage` INT(11) DEFAULT NULL,
    `ecapacity` VARCHAR(20) DEFAULT NULL,
    `transmission` VARCHAR(50) DEFAULT NULL,
    `condition` VARCHAR(50) NOT NULL,
    `numberowners` VARCHAR(20) DEFAULT NULL,
    `description` TEXT DEFAULT NULL,
    `sity` VARCHAR(100) NOT NULL,
    `numberphone` VARCHAR(20) NOT NULL,
    `owner` VARCHAR(100) NOT NULL,
    `photo` TEXT DEFAULT NULL,
    `created_at` TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP,
    PRIMARY KEY (`id`),
    INDEX `idx_mark` (`mark`),
    INDEX `idx_price` (`price`),
    INDEX `idx_release` (`release`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;

-- Таблица пользователей
CREATE TABLE `regbase` (
    `nickname` VARCHAR(256) NOT NULL,
    `email` VARCHAR(256) NOT NULL,
    `pass` VARCHAR(256) NOT NULL,
    PRIMARY KEY (`nickname`),
    UNIQUE KEY `uk_email` (`email`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
```

Индексы созданы для ускорения поиска по часто используемым полям (марка, цена, год).

```javascript
function renderCars(cars) {
    const container = document.getElementById('carsContainer');

                    <div><span class="spec-label">Пробег:</span> ${Number(car.mileage || 0).toLocaleString()} км</div>

            </div>
        `;
        container.appendChild(carElement);
    });
    
    attachPhotoHandlers();
}
```

Листинг 4.2 – Обработка загрузки файла в post.php

```php
$photo_path = null;
if (isset($_FILES['photo']) && $_FILES['photo']['error'] === UPLOAD_ERR_OK) {
    $upload_dir = 'uploads/';
    if (!file_exists($upload_dir)) {
        mkdir($upload_dir, 0777, true);
    }
    
    $finfo = finfo_open(FILEINFO_MIME_TYPE);
    $mime_type = finfo_file($finfo, $_FILES['photo']['tmp_name']);
    finfo_close($finfo);
    
    $allowed_mimes = ['image/jpeg', 'image/png', 'image/gif', 'image/webp'];
    
    if (in_array($mime_type, $allowed_mimes)) {
        $ext = pathinfo($_FILES['photo']['name'], PATHINFO_EXTENSION);
        $filename = time() . '_' . bin2hex(random_bytes(8)) . '.' . $ext;
        $photo_path = $upload_dir . $filename;
        move_uploaded_file($_FILES['photo']['tmp_name'], $photo_path);
    }
}
```

4.6. Развертывание и эксплуатация

Заключительный этап разработки — развертывание приложения:

1. Подготовка сервера (локального или удаленного):

· Установка PHP 7.4 или выше.
· Установка MySQL 5.7 или выше.
· Настройка веб-сервера (Apache или встроенный PHP-сервер).

2. Размещение файлов:

· Копирование всех файлов проекта в директорию сервера (например, C:\OpenServer\domains\myserver).
· Настройка прав доступа к директории uploads/ (чтение и запись для веб-сервера).

3. Настройка базы данных:

· Импорт структуры базы данных через phpMyAdmin или командную строку.
· Проверка подключения: изменение параметров в PHP-файлах при необходимости.

4. Конфигурация для production:

· Отключение вывода ошибок (display_errors = Off).
· Настройка HTTPS (для удаленного сервера).
· Настройка резервного копирования базы данных.

Инструкция по запуску на локальной машине:

1. Установить OpenServer (или любой другой локальный сервер).
2. Скопировать папку проекта в C:\OpenServer\domains\myserver.
3. Запустить OpenServer.
4. Импортировать базу данных из файла database.sql через phpMyAdmin.
5. Открыть браузер и перейти по адресу http://myserver/header.html.

```php
<?php
error_reporting(0);
ini_set('display_errors', 0);

header('Access-Control-Allow-Origin: *');
header('Access-Control-Allow-Methods: GET, POST, OPTIONS');
header('Access-Control-Allow-Headers: Content-Type');
header('Content-Type: application/json');

$host = 'localhost';
$db = 'projectbaseautosell';
$user = 'root';
$pass = '';

try {
    $pdo = new PDO("mysql:host=$host;dbname=$db;charset=utf8", $user, $pass);
    $pdo->setAttribute(PDO::ATTR_ERRMODE, PDO::ERRMODE_EXCEPTION);
} catch (PDOException $e) {
    echo json_encode(['error' => 'Ошибка БД: ' . $e->getMessage()]);
    exit();
}

$stmt = $pdo->query("SELECT id, mark, model, `release`, price, mileage, ecapacity, transmission, `condition`, numberowners, description, sity, numberphone, owner, photo FROM carpost");
$cars = $stmt->fetchAll();

foreach ($cars as &$car) {
    if (!isset($car['photo']) || $car['photo'] === null) {
        $car['photo'] = '';
    }
}

ob_clean();
echo json_encode($cars);
exit();
?>
```

Листинг А.2 – Функция рендеринга карточек (JavaScript)

```javascript
function renderCars(cars) {
    const container = document.getElementById('carsContainer');
    if (!container) return;
    
    container.innerHTML = '';
    
    if (!cars || cars.length === 0) {
        container.innerHTML = '<p>Нет объявлений</p>';
        return;
    }
    
    cars.forEach(car => {
        const carDiv = document.createElement('div');
        carDiv.className = 'car-item';
        
        let photoHtml = '';
        if (car.photo && car.photo !== '') {
            let photoUrl = 'http://localhost/myserver/' + car.photo;
            photoHtml = `<img src="${photoUrl}" class="car-img" data-photo-url="${photoUrl}" alt="${escapeHtml(car.mark)} ${escapeHtml(car.model)}">`;
        } else {
            photoHtml = '<div class="no-photo">📷 Нет фото</div>';
        }
        
        carDiv.innerHTML = `
            <div class="car-image">${photoHtml}</div>
            <div class="car-info">
                <h2 class="car-name">${escapeHtml(car.mark)} ${escapeHtml(car.model)}</h2>
                <div class="car-price">${Number(car.price).toLocaleString()} ₽</div>
                <div class="car-specs">
                    <div><span class="spec-label">Год:</span> ${car.release}</div>
                    <div><span class="spec-label">Пробег:</span> ${Number(car.mileage || 0).toLocaleString()} км</div>
                    <div><span class="spec-label">Коробка:</span> ${car.transmission || 'не указана'}</div>
                    <div><span class="spec-label">Состояние:</span> ${car.condition || 'не указано'}</div>
                </div>
                <div class="car-description">${escapeHtml(car.description || 'Описание отсутствует')}</div>
            </div>
            <div class="seller-info">
                👤 ${escapeHtml(car.owner || 'Продавец')} | 📞 ${car.numberphone || 'нет телефона'}
            </div>
        `;
        
        container.appendChild(carDiv);
    });
    
    attachPhotoHandlers();
}

function escapeHtml(str) {
    if (!str) return '';
    return str.replace(/[&<>]/g, function(m) {
        if (m === '&') return '&amp;';
        if (m === '<') return '&lt;';
        if (m === '>') return '&gt;';
        return m;
    });
}
```
